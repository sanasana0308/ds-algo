💻進階：Python 實作 BSTADVANCED
Python class Node: def __init__(self, val): self.val = val self.left = self.right = None def insert(root, val): if root is None: return Node(val) if val < root.val: root.left = insert(root.left, val) else: root.right = insert(root.right, val) return root
搜尋（平衡）O(log n)100萬筆只需 20 次
搜尋（不平衡）O(n)退化成鏈，跟線性搜尋一樣

👁 查看答案
def height(root): if root is None: return 0 return 1 + max(height(root.left), height(root.right))

BST 為什麼「從中間開始插」比「從小到大插」快很多？
REFERENCE ANSWER
從中間插 → 左右均衡，高度 log₂n，搜尋 O(log n)。從小到大插 → 退化成直線，高度 n，搜尋 O(n)。

B速記法 — 用「走路蓋章」來記走訪順序
🎯 先預測 對上面這棵樹做「中序走訪」，第一個輸出的數字是？
✏️ 想像你沿著樹的輪廓從左邊走一圈（像繞著樹邊走）
每個節點旁邊有 三個位置可以「蓋章記錄」：
🟣 第一次靠近節點時（左側）→ 前序 Pre-order
🔵 從左子樹回來時（下方）→ 中序 In-order
🔴 要離開節點時（右側）→ 後序 Post-order
🟣

前序（Pre-order）

順序：自己 → 左 → 右
對上面那棵樹：50 → 30 → 20 → 40 → 70 → 60 → 80
應用：複製整棵樹（父節點要先存在，才能掛上子節點）

🔵

中序（In-order）

順序：左 → 自己 → 右
結果 = 由小到大：20 → 30 → 40 → 50 → 60 → 70 → 80 ✨
應用：輸出 BST 排序結果

🔴

後序（Post-order）

順序：左 → 右 → 自己
結果：20 → 40 → 30 → 60 → 80 → 70 → 50
應用：刪除整棵樹（子節點先刪，父節點最後）

C動畫 — 走訪順序一步步亮起，並標上順序編號

🔵 中序的神奇特性對 BST 做中序走訪，輸出一定是由小到大排好序的！
🔴 後序的生活例子刪資料夾：先刪裡面的檔案，再刪資料夾本身 → 後序！

def preorder(n): if n is None: return print(n.val) ← 最前面（前序） preorder(n.left); preorder(n.right) def inorder(n): if n is None: return inorder(n.left) print(n.val) ← 中間（中序） inorder(n.right) def postorder(n): if n is None: return postorder(n.left); postorder(n.right) print(n.val) ← 最後面（後序）
🪞學習反思P2 後設認知
為什麼 BST 中序走訪輸出的一定是由小到大？
REFERENCE ANSWER
BST 定義：左子樹 < 根 < 右子樹。中序先走左（最小），再記自己，再走右（較大）。遞迴加上 BST 大小規則，輸出必然遞增！
