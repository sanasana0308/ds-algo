📖 生活比喻：猜數字遊戲

BST 每個節點就像一個「守門人」在問你：「你要找的比我大還是比我小？」
比你小 → 往左走，比你大 → 往右走，每走一步就砍掉一半可能性！
平衡的樹：問 3 次就能找完 7 個節點。不平衡：最壞要問 7 次。

B-1速記法 — 用「走路蓋章」來記走訪順序
先理解這個概念，再看 B-2 的動畫就一目了然！
✏️ 想像你沿著樹的輪廓從左邊走一圈（像繞著樹邊走）
每個節點旁邊有 三個位置可以「蓋章記錄」：
  🟣 第一次靠近節點時（左側）蓋章 → 前序
  🔵 從左子樹回來、繞過節點下方時（下方）蓋章 → 中序
  🔴 從右子樹回來、要離開節點時（右側）蓋章 → 後序

記住位置，就記住了走訪法！（用這棵樹練習：50 是根，左有 30→20/40，右有 70→60/80）
🟣

B-1-1 前序（Pre-order）— 靠近節點的瞬間（左側）蓋章

走到節點就立刻記下來，然後才去走左右子樹。
順序：自己 → 左子樹 → 右子樹
結果：50 → 30 → 20 → 40 → 70 → 60 → 80
應用：複製整棵樹（要先知道父節點，才能建子節點）

🔵

B-1-2 中序（In-order）— 從左子樹繞回、經過下方時蓋章

先把左邊全走完，回來時才記自己，然後再去右邊。
順序：左子樹 → 自己 → 右子樹
結果（BST 中序 = 由小到大）：20 → 30 → 40 → 50 → 60 → 70 → 80 ✨
應用：輸出 BST 排序結果

🔴

B-1-3 後序（Post-order）— 要離開節點時（右側）才蓋章

左右子樹都全部走完，要離開這個節點時才記下來。
順序：左子樹 → 右子樹 → 自己
結果：20 → 40 → 30 → 60 → 80 → 70 → 50
應用：刪除整棵樹（子節點先刪，父節點最後）

🔵 中序的神奇特性

對 BST 做中序走訪，輸出結果一定是由小到大排好序的！
這是 BST 最重要的性質，大學資料庫常用。

D-2為什麼要學樹？

資料結構的核心是「如何有效率地找到資料」。
陣列要一個個找（慢），BST 每次找可以砍掉一半的範圍（快）。
大學資料庫、作業系統、網路路由，幾乎所有軟體核心都用到樹——學好樹等於打通資訊的任督二脈！
