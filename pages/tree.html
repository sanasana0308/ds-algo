Python class Node: def __init__(self, val): self.val = val # 儲存數值 self.left = None # 左子節點（預設空） self.right = None # 右子節點（預設空）
② 插入（遞迴版）
def insert(root, val): if root is None: # 找到空位，插入！ return Node(val) if val < root.val: # 比根小 → 往左走 root.left = insert(root.left, val) else: # 比根大 → 往右走 root.right = insert(root.right, val) return root
③ 搜尋
def search(root, val): if root is None: return False # 走到底，找不到 if val == root.val: return True # 找到了！ if val < root.val: # 比根小，往左 return search(root.left, val) return search(root.right, val) # 比根大，往右
④ 中序走訪（輸出排序結果）
def inorder(root): if root is None: return inorder(root.left) # 先走左子樹 print(root.val) # 再印自己 → 這時輸出的就是排好序的！ inorder(root.right) # 最後走右子樹
⑤ 時間複雜度分析
搜尋（平衡樹） O(log n) 每次砍一半，n=100萬只需比 20 次
搜尋（不平衡） O(n) 退化成鏈，跟線性搜尋一樣慢
插入（平衡樹） O(log n) 找到正確位置後插入

REFERENCE ANSWER

B 速記法 — 用「走路蓋章」來記走訪順序

先理解這個概念，再看下方的動畫就一目了然！

🎯 先預測 對 BST（50為根，左30，右70）做「中序走訪」，第一個輸出的數字是？
✏️ 想像你沿著樹的輪廓從左邊走一圈（像繞著樹邊走）
每個節點旁邊有 三個位置可以「蓋章記錄」：
  🟣 第一次靠近節點時（左側）蓋章 → 前序 Pre-order
  🔵 從左子樹回來、繞過節點下方時（下方）蓋章 → 中序 In-order
  🔴 從右子樹回來、要離開節點時（右側）蓋章 → 後序 Post-order

記住位置，就記住了走訪法！（練習這棵樹：50 是根，左有 30→20/40，右有 70→60/80）
🟣

前序（Pre-order）— 靠近節點的瞬間蓋章

走到節點就立刻記下來，然後才去走左右子樹。
順序：自己 → 左子樹 → 右子樹
結果：50 → 30 → 20 → 40 → 70 → 60 → 80
應用：複製整棵樹（要先知道父節點，才能建子節點）

🔵

中序（In-order）— 從左子樹繞回、經過下方時蓋章

先把左邊全走完，回來時才記自己，然後再去右邊。
順序：左子樹 → 自己 → 右子樹
結果（BST 中序 = 由小到大）：20 → 30 → 40 → 50 → 60 → 70 → 80 ✨
應用：輸出 BST 排序結果

🔴

後序（Post-order）— 要離開節點時才蓋章

左右子樹都全部走完，要離開這個節點時才記下來。
順序：左子樹 → 右子樹 → 自己
結果：20 → 40 → 30 → 60 → 80 → 70 → 50
應用：刪除整棵樹（子節點先刪，父節點最後）

🪞 學習反思 P2 後設認知
為什麼 BST 中序走訪輸出的一定是由小到大的順序？試著用 BST 的定義來解釋。
REFERENCE ANSWER
BST 的定義是左子樹 < 根 < 右子樹。中序走訪先走左子樹（最小值），再記錄自己，再走右子樹（較大值）。這個「先左、再自己、再右」的順序，加上 BST 的大小規則，就保證了輸出一定是遞增排序！
