Linked List（鏈結串列）

🗺️ 尋寶遊戲比喻每個節點帶著「下一個在哪」，插入新節點只要改指標，不用搬別人。
❓ LL 也有「第幾個」的概念嗎？
有的，但 LL 沒有捷徑（沒有 index 公式）。
要找第 n 個節點，必須從 head 走 n 步（O(n)）。
所以 LL 的「插入/刪除」實際上是：先走 O(n) 找位置，再花 O(1) 改指標。
LL 的優勢是「改指標那一步不需要搬移其他節點」，Array 則是每個元素都要搬。

💭 反思一下 後設認知練習
操作完之後，試著用自己的話回答：
Array 和 Linked List 在「插入」這件事上，根本差異是什麼？
REFERENCE ANSWER
Array：必須把後面所有元素物理搬移一格，時間複雜度 O(n)。
Linked List：只修改前後兩個節點的指標，本身 O(1)；但需先從 head 走到位置，O(n)。
根本差異：Array 慢在「搬移資料」；Linked List 慢在「尋找位置」，兩者慢的原因不同。

看完效能比較，用一句話總結：什麼情況選 Array，什麼情況選 Linked List？
REFERENCE ANSWER
選 Array：頻繁隨機存取、資料量已知、希望省記憶體。
選 Linked List：頻繁插入/刪除、資料量動態變化。
口訣：「讀多選 Array，改多選 Linked List。」

💡 記憶體到底長什麼樣？Array 的資料住在「連棟公寓」——地址是連續的，CPU 找起來超快。
Linked List 的資料可能住在城市各地——每棟還要貼一張「下一棟在哪」的紙條。

💭反思一下後設認知練習
從記憶體角度解釋：為什麼 Array 能「一步」存取第 i 個，而 Linked List 必須「一步一步走」？
REFERENCE ANSWER
Array：元素連續排列，大小固定，用公式 起始+index×大小 直接計算位址，O(1)。
Linked List：節點散落各處，只能從 head 逐一跟隨指標，無法跳過，O(n)。
根本原因：Array 靠「數學計算」；Linked List 靠「逐節跟隨指標」。
